## Sobre este portfólio

Este repositório apresenta minha trajetória acadêmica e profissional durante os quatro primeiros semestres do curso de **Análise e Desenvolvimento de Sistemas** da **Fatec São José dos Campos - Prof. Jessen Vidal**.

O objetivo deste Trabalho de Graduação 1 é demonstrar, por meio dos projetos de **Aprendizagem por Projetos Integrados (API)**, como desenvolvi competências técnicas, comportamentais e de gestão ao longo do curso. A proposta não é apenas listar tecnologias utilizadas, mas mostrar minha evolução como desenvolvedor, minha atuação em equipes ágeis, meu protagonismo nos projetos e minha capacidade de transformar problemas reais em soluções tecnológicas aplicáveis.

Ainda continuarei no curso e realizarei mais dois APIs nos próximos semestres. Portanto, este portfólio representa uma etapa intermediária da minha formação, consolidando os aprendizados construídos até o 4º API e servindo como base para a evolução dos projetos seguintes.

## Escala de proficiência

Para indicar meus níveis de conhecimento, utilizo uma adaptação da **Taxonomia de Bloom**, conforme a escala abaixo:

| Nível | Descrição |
|---|---|
| Ouvi falar | Tive contato inicial com o tema e reconheço sua finalidade |
| Entendi | Compreendo os conceitos principais e consigo explicar sua aplicação |
| Sei fazer com ajuda | Consigo aplicar o conhecimento com apoio, consulta ou orientação |
| Sei fazer com autonomia | Consigo aplicar o conhecimento de forma independente em contextos reais |

Como já ingressei na Fatec sabendo programar e com base prática em tecnologia, classifiquei as tecnologias exercitadas nos APIs como **Sei fazer com autonomia**. A principal exceção é o **Jira**, ferramenta que utilizei no contexto dos projetos, mas que ainda não considero dominar plenamente.

## Evolução durante o curso

Ao longo dos quatro primeiros APIs, minha evolução ocorreu de forma progressiva, mas sempre com uma postura ativa. No primeiro semestre, além de desenvolver páginas para o projeto Horus, assumi o papel de Scrum Master, organizando o time e ajudando a transformar o conteúdo sobre metodologia ágil em uma aplicação web funcional.

No segundo semestre, passei a atuar de forma mais técnica como integrante do time de desenvolvimento, contribuindo com estrutura, telas, lógica de negócio e funcionalidades em um sistema voltado à extração de informações de documentos. Esse projeto ampliou minha compreensão sobre aplicações com banco de dados, interface, regras de negócio e uso de modelos de linguagem e visão.

No terceiro semestre, vivi a experiência mais marcante da minha trajetória até o momento. No AdaTrade, atuei como Product Owner, desenvolvedor backend principal e responsável pelo processo de ETL dos dados. Esse API exigiu maturidade técnica e organizacional, pois envolveu grandes volumes de dados oficiais, construção de API REST, integração com frontend e comunicação direta com o cliente para definição do escopo, priorização e validação das entregas.

No quarto semestre, aprofundei minha atuação em backend e inteligência artificial aplicada. No AdaTech, tive protagonismo na estrutura do sistema, na criação de regras de negócio e na implementação de recursos centrais envolvendo **RAG**, **Ollama** e automação da geração de instruções de registro aduaneiro. A experiência exigiu atenção a dados sensíveis, precisão técnica, integração entre IA e sistema web, além de cuidado com conformidade e revisão humana.

Essa sequência mostra uma evolução clara: comecei organizando equipe e construindo páginas, avancei para implementação de funcionalidades completas, depois assumi responsabilidades de produto, dados e backend, e finalmente passei a atuar em soluções mais complexas envolvendo IA, automação e regras de negócio críticas.
